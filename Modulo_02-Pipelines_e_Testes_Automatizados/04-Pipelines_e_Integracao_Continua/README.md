# DevSecOps - Aula 2.4: Pipelines e Integração Contínua com GitLab

## 🎯 Objetivo da Atividade
Configuração prática de um pipeline de CI/CD (Integração Contínua e Entrega Contínua) utilizando o GitLab para automação de build, testes e deploy de uma aplicação em Go (Golang).

## 🛠️ Tecnologias e Conceitos Utilizados
* **GitLab CI/CD:** Automação de esteiras de desenvolvimento.
* **GitLab Runner:** Agente responsável por executar os jobs do pipeline.
* **Docker & Docker Compose:** Criação de ambiente isolado para o Runner e aplicação.
* **Go (Golang):** Linguagem de programação da aplicação de exemplo.

## 📋 Etapas do Laboratório
- [x] Passo 1: Preparação do Ambiente e Registro do GitLab Runner
- [ ] Passo 2: Preparação do Build (Dockerfile e código Go)
- [ ] Passo 3: Configuração do Pipeline Básico (.gitlab-ci.yml)
- [ ] Passo 4: Configuração do Deploy Automatizado
- [ ] Passo 5: Monitoramento do Pipeline
